## Notes

### Architecture Decisions
- **Durable Objects**: Used for long-running tasks (exceeding Worker 10s limit) with checkpointing every 3 tool calls
- **Tool Caching**: Parallel-safe tools (read-only) are cached; mutation tools run sequentially
- **Model Routing**: Tasks are categorized (coding/reasoning/general) and routed to appropriate free/paid models
- **Progressive Disclosure**: Simple queries skip planning phase; complex queries use structured plan/work/review phases

### Risks & Mitigations
- **Large File Size**: `src/durable-objects/task-processor.ts` (149KB) is difficult to maintain — addressed in Phase 1
- **Test Coverage**: Worker entrypoint has minimal tests — addressed in Phase 2
- **External Dependencies**: Tool calls depend on third-party APIs (GitHub, OpenRouter) — handled with retries and rate limiting

### Open Questions
- Should we add a dedicated orchestration service (separate from DOs)?
- How to handle stateful user sessions across container restarts?
- Should we add a rate-limiting layer for API calls?
